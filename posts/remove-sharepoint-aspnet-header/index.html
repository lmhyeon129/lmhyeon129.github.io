Remove SharePoint ASP.NET Header

Remove SharePoint ASP.NET Header

보안 상 시스템 정보가 HTTP Header에 노출되는 것을 최소화 하세요. 의 권고로 시작된 작업

대상 목록

  • X-SharePointHealthScore
  • Server
  • SPRequestGuid
  • X-AspNet-Version
  • X-Powered-By
  • MicrosoftSharePointTeamServices
  • X-MS-InvokeApp

X-SharePointHealthScore

  • 값은 0~10의 숫자

참고: https://msdn.microsoft.com/en-us/library/jj162162(v=office.12).aspx

The server can monitor the current load and its ability to process requests. If it monitors the load, it can return the load information to the client in a X-SharePointHealthScore header. This header specifies a value between 0 and 10, where 0 represents a low load and a high ability to process requests and 10 represents a high load and that the server is throttling requests to maintain adequate throughput.

sharepoint 서버의 상태를 0부터 10까지 표현하는 숫자 입니다.
이런 숫자마저 보안상 문제가 있다고 한다면, 숨깁니다.

이 header는 설정으로는 숨길 수가 없고, custom Module을 만들어서 iis 에 등록을 해야 합니다.
Init 시점에 PreSendRequestHeaders 에 해당 header를 삭제하는 처리기를 만들어서 추가해 줍니다.

Server

  • 값은 IIS 버전

SPRequestGuid

  • 값은 Guid 문자열

참고: https://msdn.microsoft.com/en-us/library/dd908737(v=office.12).aspx

ULS log에서 해당 request guid로 찾기 위해서 기록한다고 합니다.

X-SharePointHealthScore, Server, SPRequestGuid 헤더들은 IIS URL Rewrite module을 설치해서 처리할 수도 있고, 위에서 나온 custom Module을 만들어서 처리할 수도 있습니다.

IIS URL Rewrite module 통한 header 값 삭제

  1. 서버변수에 RESPONSE_{{HEADER이름}} 추가 (전부 대문자로)

  2. 아웃바운드 규칙 추가

    1. 검색 번위: 서버 변수
    2. 변수 이름: 위에 추가한 변수
    3. 변수 값: 패턴과 일치, 사용: 정규식
    4. 패턴: .* (모든거 다)
    5. 작업 유형: 재작성
    6. 값: 빈칸 (기존 서버 변수 값 바꾸기 체크)

X-AspNet-Version

  • 값은 ASP.NET Version (2.0.xxx)

web.config 파일에 system.web > httpRuntime 설정을 찾아서 enableVersionHeader="false" 속성을 추가합니다.
하위에 응용 프로그램에서 별도로 true로 설정하지 않으면 상속을 받아서 header 숨김이 처리 됩니다.

<configuration>
  <system.web>
    <httpRuntime enableVersionHeader="false" />
  </system.web>
</configuration>
# 설정하는 script
appcmd.exe set config "경로이름" -section:system.web/httpRuntime /enableVersionHeader:"False"

X-Powered-By

  • 값은 ASP.NET, PHP 등등 기술의 이름 중 하나

IIS 관리자 > HTTP 응답 헤더에서 삭제 가능

또는, web.config에서 아래와 같이 처리하면 됩니다.

<configuration>
	<system.webServer>
  	<httpProtocol>
    	<customHeaders>
      	<remove name="X-Powered-By" />
      </customHeaders>
    </httpProtocol>
  </system.webServer>
</configuration>

MicrosoftSharePointTeamServices

참고: https://wmostafaw.com/2013/08/31/removing-sharepoint-2013-version-form-http-header/

NOTE: this may effect other applications like InfoPath.

SharePoint 버전을 표기하는 header로, 이 header가 없으면 infopath에서 sharepoint 연계된 작업을 할 때 영향을 받습니다.

숨기는 방법은 X-Powered-By 와 동일하게 HTTP 응답 헤더에서 삭제 가능하고, web.config 에서 remove 노드를 추가하는 방법으로도 가능합니다.
