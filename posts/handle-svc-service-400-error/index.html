svc 서비스 400 error 처리

svc 서비스 400 error

svc 서비스에서 400 에러가 나서 그것에 대한 원인 찾는 과정

.svc 를 가지고 서비스 참조에 추가해서 호출하는데 400 error가 나서 원인을 검색

요청 메시지 허용크기 초과

대부분 case가 요청 메시지의 용량 초과에 대한 case로 나타남 이는 다음과 같이 MaxReceivedMessageSize 값 과 ReaderQuotas 값을 조정하여 해결할 수 있다

 using (CommonBoardService.CommonBoardServiceClient client = new CommonBoardService.CommonBoardServiceClient(
    new System.ServiceModel.WebHttpBinding()
    {
      MaxReceivedMessageSize = 1024000000, 
      ReaderQuotas = new System.Xml.XmlDictionaryReaderQuotas() { MaxStringContentLength = 2147483647 },
    }, new EndpointAddress(new Uri(uri))
))
{
    // BoadyStyle : Wrapped 로 설정
    client.Endpoint.Behaviors.Add(new WebHttpBehavior() { DefaultBodyStyle = System.ServiceModel.Web.WebMessageBodyStyle.Wrapped });
    // 여기서 호출
}

처리하지 못한 예외 (+ svc trace log 설정)

하지만 이걸로도 해결이 되지 않는 case가 있어서 로그를 남길 수 있는 설정에 대해서 검색 현재는 오류가 나면 catch문에서 txt file에다가 에러로그를 쓰게끔 되어 있다. 이렇게 되면 에러를 처리했기 때문에 200ok 로 response 되지만, response data에 error내용과 코드를 심어서 전달

하지만 400 error는 catch에서 처리하지 못한 case이기 때문에, 요청 전인지, 요청 처리하는 중인지 알 수 가 없었음.

서버에서 svc에 대한 로그를 남기는 방법은 web.config에 아래 노드를 추가

<system.diagnostics>
    <sources>
        <source name="System.ServiceModel"
                  switchValue="Information, ActivityTracing"
                  propagateActivity="true">
            <listeners>
                <add name="traceListener"
                     type="System.Diagnostics.XmlWriterTraceListener"
                     initializeData= "D:\logs\Traces.svclog" />
            </listeners>
        </source>
    </sources>
</system.diagnostics>

그렇게 하여 svc 요청을 보내 400error를 한번 받으면 아래와 같은 trace log가 파일로 남게 된다 <E2ETraceEvent> 이 노드들이 있고 하위 <ApplicationData> 에 상세 내용이 들어 있음

예외가 났고, 예외를 처리하지 않아 이렇게 되어 있는것을 확인. 확인결과 try-catch 밖에서 해당 예외가 발생한 것을 확인
